# 🎯 Day 11: Blackjack Capstone Project

## 🃏 Project Overview

Today I built a terminal-based **Blackjack game** in Python, inspired by the classic casino card game. The player competes against the computer, making decisions to draw additional cards or stand while trying to reach a score as close to 21 as possible without going over.

---

## 🚀 What I Learned

* Creating and using custom functions
* Working with lists to manage card hands
* Using loops to control game flow
* Implementing conditional logic for game decisions
* Generating random outcomes with the `random` module
* Handling edge cases and special game rules
* Structuring a larger program into smaller reusable components

---

## ✨ Features Implemented

✅ Random card dealing

✅ Blackjack detection

✅ Automatic Ace value adjustment (11 → 1)

✅ User hit/pass functionality

✅ Computer decision-making logic

✅ Win, Lose, and Draw outcomes

✅ Replay game option

---

## 🧠 Challenges Faced

* Implementing Blackjack rules correctly
* Managing Ace values when scores exceeded 21
* Determining when the game should end
* Comparing scores while handling special cases like Blackjack and busts

---

## 💡 Key Takeaway

This was my first larger Python project where multiple concepts came together in a single application. I initially attempted the project independently before comparing it with the reference solution, which helped me identify improvements in code organization, modularity, and handling of edge cases.
